# Windows-Native Environment Setup Notes

The project will now use a native Windows route.

Conda will remain useful for Python metadata work, but not for the core aligner. `minimap2` is not available from Bioconda for `win-64`, and `samtools` is also not distributed by Bioconda for `win-64`.

The Windows-native replacement route is:

- SRA Toolkit for download and FASTQ conversion.
- NCBI Magic-BLAST for human/mouse sequence-level QC.
- BLAST `makeblastdb`, bundled with Magic-BLAST/BLAST+, for hg38/mm10 databases.

### 2. Required Windows Tools

| Tool | Provides | Why needed |
|---|---|---|
| `sra-tools` | `prefetch`, `fasterq-dump`, `vdb-dump`, `vdb-validate` | Download SRA data and convert SRA to FASTQ |
| `magicblast` | `magicblast` | Map reads to hg38 and mm10 for species QC on Windows |
| `makeblastdb` | `makeblastdb` | Build searchable BLAST databases from `hg38.fa` and `mm10.fa` |

### 3. Reference Files

These are required before species QC:

- `D:\二区\hg38.fa`
- `D:\二区\mm10.fa`
- `D:\二区\blastdb\hg38.*`
- `D:\二区\blastdb\mm10.*`

The `.fa` files are reference genomes. The `blastdb` files are generated by `makeblastdb`.

## Build Magic-BLAST Databases

After `hg38.fa` and `mm10.fa` are placed in `D:\二区`, run:

```bat
cd /d D:\二区
mkdir blastdb
makeblastdb -in hg38.fa -dbtype nucl -parse_seqids -out blastdb\hg38 -title hg38
makeblastdb -in mm10.fa -dbtype nucl -parse_seqids -out blastdb\mm10 -title mm10
```

## First Pilot: hMM1 Species QC

Pilot sample:

```text
GSM8329293 / hMM1 / SRX24927515
```

Download and convert to FASTQ:

```bat
cd /d D:\二区
mkdir sra_cache fastq tmp_fasterq qc_species_hMM1

prefetch SRX24927515 --output-directory sra_cache --max-size 100G

fasterq-dump SRX24927515 ^
  --split-files ^
  --outdir fastq ^
  --temp tmp_fasterq ^
  --threads 8 ^
  --progress ^
  --disk-limit 300GB
```

Map to human:

```bat
magicblast ^
  -query fastq\SRX24927515_1.fastq ^
  -query_mate fastq\SRX24927515_2.fastq ^
  -db blastdb\hg38 ^
  -infmt fastq ^
  -outfmt tabular ^
  -num_threads 8 ^
  -out qc_species_hMM1\hMM1_hg38_magicblast.tsv
```

Map to mouse:

```bat
magicblast ^
  -query fastq\SRX24927515_1.fastq ^
  -query_mate fastq\SRX24927515_2.fastq ^
  -db blastdb\mm10 ^
  -infmt fastq ^
  -outfmt tabular ^
  -num_threads 8 ^
  -out qc_species_hMM1\hMM1_mm10_magicblast.tsv
```

Expected direction:

- hg38 hit/read count should dominate.
- mm10 hit/read count should be very low.

## Why This Is Acceptable For Windows

This changes the first-pass sequence QC method from `minimap2 + samtools` to `Magic-BLAST`.

This is acceptable for the pilot because:

- The QC question is simple: human-dominant vs mouse-dominant mapping.
- Magic-BLAST is an NCBI aligner with official Windows binaries.
- It accepts FASTQ paired reads and outputs SAM or tabular results.

In the Methods, describe this as:

> Sequence-level species assignment was performed using Windows-native NCBI Magic-BLAST against separate human and mouse reference databases. Samples were retained in the human mainline only when human alignment evidence dominated and mouse alignment evidence remained negligible.
